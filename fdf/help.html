KiWQM Field Data Importer Help

Introduction

Welcome to the KiWQM Field Data Importer Help file. This document briefly describes how to use the Field Data Importer, provides explanations of the various fields requiring user input, and defines the codes used.

How to use this application

KiWQM requires that any imported data is loaded in a specific format. The KiWQM Field Data Importer helps get your field data into that format easily. The application can be used to load data from a data logger file (such as those created by the YSI or Hydrolab instruments when measurements are saved in the field) or to input data collected manually on a Water Sample Log.

The application consists of a single screen with access to all the required functions.

Freezing columns

Data entry requires entering data into many columns (up to 34 columns) so it is usually necessary to scroll to the right to view, validate and enter data. In order to keep track of the samples being edited, it is possible to freeze a number of columns on the left of the table so that they remain visible through the entire editing process. The Frozen columns box at the right side of the window sets the number of columns at the left that will remain visible. The default is 3 columns, which allows for the measuring program, station and date to be visible at all times. You can increase or decrease the number of frozen columns by using the up and down arrows provided, or by typing a number directly into the box.

Manual data entry

To input data manually from a log sheet, just press the Add lines button (once press for each sample required) and start entering data in the table. See Description of fields for descritions of the various fields, and details on mandatory and optional fields.

Importing a data file from a field instrument

To import a data file created by a Hydrolab or YSI instrument, do the following:

  1. Select the instrument from the Instrument drop down list at the top of the window.
  2. Press the "..." key labeled File location.
  3. Select the file you wish to import.
  4. Select the date format of the file from the drop-down list. You can use the Swap day and month button later on to correct this if you make a mistake at this point (see Editing functions below for more information).
  5. Press the Add file button.

The data will be automatically imported from the selected file and the file name will be displayed in the Current files box. You will be notified if there are problems with the data or if the data file is invalid for the selected instrument.

Adding additional data or files

The user is not restricted to opening a single file at once. To open additional files, follow the procedure outlined in Importing a data file from a field instrument above. The newly added data will be added at the bottom of the already displayed data, and the new file name will be added to the list in the Current files box.

You can also add additional data manually by pressing the Add lines button and following the steps in Manual data entry.

Editing data

Manual editing

Data can be edited directly in the table. Some columns require selection of values from a drop-down list (see Description of fields for more information).

To edit values that do not require selection from a drop-down list, select the cell and begin typing. Once you are done (by pressing enter, tab, or one of the arrow keys, or by selecting another cell with the mouse), the application will check if the value added is valid and will report a message if it is not.

You will not be able to type in completely invalid values. For example, if you try to type 20 into the pH field, you will be able to type 2 but the application will prevent you from typeing the 0 because 20 is an invalid value for pH.

To edit values in a drop down list, double-click in the cell you wish to edit and the drop-down list will appear. Alternatively, if you know all or part of the value you wish to select, simply select the cell (by clicking once in the cell, or by navigating to it with the arrow keys) and begin typing - a filtered list of options will appear and you can make your selection from the list.

Editing functions

Several buttons have been supplied that provide convenience editing functions:

  • Fill location and sequence: The majority of the time, water quality samples collected will have a location number of 1 and a sequence number of 1 (for more information on these, see Water quality - users' guide to water sample logs (STOP 32012)). To ease data entry, a the Fill location and sequence button has been provided that will fill the location number and sequence number columns of any selected rows with ones. The validation checks will determine if any location or sequence numbers are invalid before export.
  • Swap day and month: For instances when the month and day have been accidentally reversed (e.g. the date has been recorded in the American format MM/DD/YYYY but has been interpreted in the Australian format DD/MM/YYYY), this button will swap the digits for day and month in the selected dates. This will also update the Sampling ID. Please note that the cells with the dates to be swapped must be selected.

Copying and pasting

You can copy and paste data between cells and also between the FDF application and other programs such as Excel. After selecting the range to copy or paste, copy and paste inside FDF can be accomplished by either of the following methods:

  1. Right-click and select copy or paste from the menu as appropriate.
  2. Using the keyboard shortcuts Ctrl-C (copy) and Ctrl-V (paste)

Deleting data

To delete an entire row (or rows), select the row(s) by pressing the row number at the left, and then press the Delete selected key. Lines can be added in the data set by selecting a row (or rows) and pressing the Add lines button - a number of blank lines will be added at the selected location.

You can reset all data by pressing the Reset data button.

Managing not-visited or not-collected items

Sometimes a station is not visited or a sample not collected if conditions are not appropriate. In these cases it is usually a requirement to log the scheduled sample without any results.

The following fields are required to be filled when a station is not visited or a sample not collected:

  • Measuring program number
  • Station number
  • Date
  • Sampling officer
  • Station visited
  • Sample collected
  • Comments

The comment should contain a description of the reason for which the station was not visited or the sample not collected.

Exporting data

Once you have completed the data entry, press the Export data button to export the data set in a KiWQM-friendly format. The application will check that all required columns are complete and that all invalid values have been rectified. You will not be able to export the data with required columns that are incomplete, or with invalid values in the data set.

An additional "sample-oriented" file can be exported at the same time by ticking the Export sample-oriented file checkbox. This additional file can be opened in Excel or any text editor and used for QA purposes. Additionally, the contents of this file can be pasted directly back into the application if corrections need to be made later on.

Note: The "sample-oriented" file cannot be imported to KiWQM.

After the data has been exported, it is ready for importing to KiWQM. See the procedure Bulk Data Import into KiWQM Database (STOP 32055) for details on this procedure.

Description of sample metadata fields

Field Definition Mandatory

MP# (Measuring program number)

The measuring program number assigned to the project in KiWQM. This is in the form MP###.

Y

Station#

A 6-8 digit identifier that specifies the monitoring station at which the sample was collected. See procedure Water quality project, sample, fraction and station numbers (product identification and traceability) (STOP 32005) for further information.

Y

Date

The date the sampling event took place. The date can be input in any format as long as it contains two digits for day, two digits for month and two or four digits for year in that order. The date will be automatically reformatted to the format DD/MM/YYYY if it was not entered as such.

Y

Time

The time at which the field measurements were taken or the sample was collected. The time can be entered in any format as long as it is entered in 24-hour time and contains at least two digits for hour and two digits for minute. The time can include seconds (two digits). The time will be automatically reformatted to the format HH:MM:SS (24-hour time). In general, the seconds should be zero so as to match the data exported by the laboratory. Where field measurements are taken using a data logger, the time recorded on the WSL should match the time recorded on the data logger. Since samples are often collected and bottled just after the field measurements are taken, the time recorded here may be slightly earlier than the exact time the sample was removed from the media or bottled.

The time recorded here must match the time recorded on the Water Sample Log filled at time of sample collection.

Y

Sampling ID

The sampling ID is a unique identifier that is used to group samples collected together as part of a single sampling event. To ensure uniqueness, it is generated from the station number and the date in the format STATION_NUMBER-DDMMYY. These fields are described in more detail in STOP 32005: Water quality project, sample, fraction and station numbers (product identification and traceability).

Individual samples within the sampling are identified in conjunction with the location number and sample sequence number as described below. In most situations, a single sample will have a single Sampling ID.

Y (automatically generated)

Location# (Location number)

A single-digit number used to separate samples from the same station and sampling event, but collected from different spatial locations. This is mostly relevant for groundwater samples where there could be multiple pipes at a single site, but is also relevant for surface water samples where spatial analysis of water quality at a particular station is important.

The location number starts at 1 and increments by one for each new location sampled. For groundwater samples, this column is used to indicate the pipe number. If only a single location or pipe exists at a station, this should be filled with a 1.

Y

Sequence# (Sample sequence number)

A single-digit number used to specify individual samples within a sampling event at a given location. Every sample collected must have a sequence number associated with it. The sequence number starts at 1 and increments by one for each sample collected. For example, if three samples are collected at a station at different depths to form a depth profile, the first sample will have a sequence number of 1, the second sample will have a sequence number of 2, and the third sample will have a sequence number of 3. This also applies to replicate samples collected at a site. For example, a primary sample may have a sequence number of 1, while the replicate sample will have a sequence number of 2. If only one sample is collected at a station, it should be assigned a sequence number of 1.

Y

Matrix

A two-letter code that describes the medium from which the sample was collected. This field must be populated with one of the acceptable values from the controlled list. Code definitions are provided in Code definitions below.

Y

Collection method

Indicates the method by which the sample was collected. This field must be populated with one of the acceptable values from the controlled list. Code definitions are provided in Code definitions below.

Y

Sample type

Indicates whether the sample is a primary sample or some kind of quality control sample. This field must be populated with one of the acceptable values from the controlled list. Code definitions are provided in Code definitions below.

Y

Calibration record

Instrument calibrations should be recorded using the appropriate Form 39004: Instrument calibration log and maintenance schedule, according to the Instrument calibration log user guide (STOP 32057). The number to be recorded here is the Instrument book number and Sheet number combined, but separated by a forward slash. For example, if the calibration log for an instrument had an Instrument book number of BA-0123, and a Sheet number of 02, then the calibration record to be noted is BA-0123/02. This provides the opportunity to trace back from the collected field data to the original calibration record if required. If more than one calibration record is relevant, note all the relevant record numbers.

N

Instrument

The model of water quality instrument used to collect the field measurements. This must be selected from the list of available instruments.

Y

Turbidimeter

The model of turbidity instrument used to collect the turbidity measurements. This must be selected from the list of available instruments.

N

Sampling officer

Name of the staff member who collected the field measurements.

Y

Station visited

Indicates whether the station was visited (YES) or if this entry is for a scheduled visit that did not take place (NO).

N

Sample collected

Indicates whether a sample has been collected for laboratory analysis, or whether only field measurements have been collected. This is indicated by selecting either YES or NO.

N

Depth Upper (m)

If a sample is collected at a specific depth, write the depth in metres.

Y

Depth Lower (m)

If the sample is collected over a depth range, record the upper depth limit in Depth Upper and lower depth limit in Depth Lower.

N

Comments

Any relevant text comments that should accompany the data.

N
